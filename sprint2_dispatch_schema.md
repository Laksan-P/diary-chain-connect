# Sprint 2 Database Configuration

To store the Dispatch and Dispatch Item details in your Supabase database, you need to run the following SQL script to create the necessary tables.

## Instructions
1. Open your [Supabase Dashboard](https://supabase.com/dashboard).
2. Select the `dairy-chain-connect` project.
3. On the left sidebar, click on **SQL Editor**.
4. Click **New Query**.
5. Copy and paste the SQL code below into the editor.
6. Click **Run**.

## SQL Script

```sql
-- 1. Create the dispatches table
CREATE TABLE IF NOT EXISTS dispatches (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  chilling_center_id BIGINT REFERENCES chilling_centers(id) ON DELETE CASCADE,
  transporter_name TEXT NOT NULL,
  vehicle_number TEXT NOT NULL,
  driver_contact TEXT NOT NULL,
  dispatch_date TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()),
  status TEXT DEFAULT 'Pending' CHECK (status IN ('Pending', 'Dispatched', 'Approved', 'Rejected')),
  rejection_reason TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now())
);

-- 2. Create the dispatch items table
CREATE TABLE IF NOT EXISTS dispatch_items (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  dispatch_id BIGINT REFERENCES dispatches(id) ON DELETE CASCADE,
  collection_id BIGINT REFERENCES milk_collections(id) ON DELETE CASCADE
);

-- 3. Add dispatch_status to milk_collections if it doesn't exist
ALTER TABLE milk_collections 
ADD COLUMN IF NOT EXISTS dispatch_status TEXT DEFAULT 'Pending' CHECK (dispatch_status IN ('Pending', 'Dispatched', 'Approved', 'Rejected'));
```
